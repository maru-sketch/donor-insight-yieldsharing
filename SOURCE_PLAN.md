# 후원자 데이터 인사이트 대시보드

## 기획서 메타데이터
- source: orange-build-app
- contract_version: 2
- deliverable_kind: web_app
- 추천 결과물 형태: 웹앱

> 후원자 납부 및 인입 데이터를 업로드하면 AI가 기부 패턴과 전략적 인사이트를 요약해 주는 모금 담당자용 대시보드

## 누구의 어떤 문제인가요?
- 대상 사용자: 열매나눔재단 모금 담당자
- 해결할 문제: 후원자 데이터를 매번 수동으로 정리하고 분석해야 하며, 전략 수립을 위한 핵심 인사이트 도출에 많은 시간이 소요됨
- 가장 막히는 순간: 후원 요청 전략을 수립하기 위해 내부 시스템 데이터를 직접 조회하고 정리하는 과정에서 업무 시간 지체 및 분석 누락 발생

## 해결 방식
후원자 엑셀 데이터를 업로드하면 AI가 납부 항목과 인입 채널별 패턴을 분석하여 모금 전략 리포트를 자동 생성하는 웹앱

## 사용 흐름
1. 후원자 데이터 엑셀/CSV 파일 업로드
2. AI의 납부 및 인입 채널별 패턴 자동 분석
3. 대시보드 상에서 AI가 제안한 모금 전략 리포트 확인
4. 담당자가 리포트 내 핵심 인사이트 문구 수정 및 저장

## 핵심 기능
- CSV 기반 후원자 데이터 자동 시각화
- AI 기반 기부 패턴 및 인사이트 분석 리포트
- AI 생성 인사이트 문구 편집 및 커스텀 리포트 저장 기능

## 첫 구현의 범위
### 포함
- CSV 데이터 파싱 및 시각화
- AI 기반 요약 리포트 생성
- 데이터 수정 및 리포트 내보내기

### 제외
- 내부 후원 관리 시스템과의 실시간 API 연동
- 개별 후원자 맞춤형 이메일 발송 자동화
- 사용자 권한 관리 및 계정별 로그 분석

## 사용할 자료
후원자 성명, 연락처 등 민감 정보가 포함된 엑셀/CSV 파일을 업로드하므로, 데이터 분석 후 서버에 저장하지 않고 휘발성으로 처리하거나, 파일 업로드 시점에 마스킹 처리 가이드를 적용함

## AI와 사람의 역할
- AI가 할 일: 업로드된 데이터를 바탕으로 납부 항목별 금액 추이와 인입 채널별 기부 빈도를 분석하여, 다음 후원 요청 전략 수립을 위한 핵심 요약 문구 생성
- 사람이 확인할 일: AI가 생성한 분석 결과 중 후원 전략에 직접적인 영향을 미치는 핵심 인사이트 문구의 톤앤매너와 정확성을 담당자가 검토 후 수정

## 성공 기준
- 데이터 업로드 후 1분 이내 시각화 차트 표시
- AI가 생성한 텍스트를 사용자가 직접 수정하여 저장 가능
- 모금 전략 수립 시간을 이전 대비 50% 이상 단축

## 실패 신호
- CSV 파일 형식 불일치로 인한 파싱 오류
- AI가 생성한 인사이트 문구가 의미 없이 반복됨
- 데이터 시각화 차트가 모바일 환경에서 깨짐

## 직접 확인할 3가지

### TEST-01 — 정상적인 데이터 업로드 및 인사이트 생성 확인
- 준비: 기부 패턴이 포함된 정상적인 CSV 파일 준비
- 행동: 웹앱에 CSV 파일 업로드
- 기대 결과: 시각화 차트와 AI가 생성한 인사이트 문구가 화면에 표시됨
- 통과 증거: 화면에 표시된 차트 이미지와 AI 요약 텍스트 캡처

### TEST-02 — 잘못된 파일 형식 처리 확인
- 준비: 기부 데이터가 아닌 무관한 텍스트 파일 준비
- 행동: 잘못된 파일 업로드 시도
- 기대 결과: 오류 메시지 발생 및 서비스 중단 없이 재입력 유도
- 통과 증거: 오류 메시지가 포함된 브라우저 화면 캡처

### TEST-03 — AI 생성 인사이트 문구 편집 기능 확인
- 준비: AI가 생성한 인사이트 문구가 화면에 표시된 상태
- 행동: 텍스트 박스를 선택하여 문구를 수정하고 저장 버튼 클릭
- 기대 결과: 수정된 문구가 대시보드에 반영되어 저장됨
- 통과 증거: 수정 전/후 화면 비교 캡처

## 바로 시작할 순서
1. CSV 파일 업로드 UI 구현
2. 업로드된 데이터의 기본 통계 시각화(차트) 구현
3. AI가 차트를 해석하여 요약 문구를 출력하는 프로토타입 연결

## 가정 및 확인할 점
- 모금 담당자는 정기적으로 CSV 형태의 데이터를 추출할 수 있는 권한이 있음
- 현재 보유한 데이터에는 납부 항목과 인입 채널 컬럼이 명확히 존재함
